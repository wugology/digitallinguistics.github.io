    Introduction

    This document lays out a standard format for exchanging linguistic data in digital form, which we call dlxJSON format (short for Digital Linguistics JavaScript Object Notation). The format is designed to capture hierarchical linguistic data in a way that aligns with the descriptive categories that linguists use, relying on fundamental linguistic notions such as text, morpheme, orthography, etc. For instance, it is capable of capturing the fact that a text contains sentences, sentences contain words, words contains morphemes, and morphemes contain phonemes. This turns out to be a crucial factor in inputting, editing, searching, and analyzing linguistic data. For a more detailed discussion of the advantages of this format in doing Digital Linguistics, see Hieber & Hall (forthcoming).

    There are many ways that someone could digitally represent linguistic data. For example, one could represent the ‘language’ field as "lang" or "language". While either is equally appropriate, portability demands that linguist standardize on a single format. (Note that the current document is a working version, and so many parts of this specification are still subject to change.) The current specification provides a standard for how linguists can represent their data using JSON, i.e. JavaScript Object Notation. JSON is the data interchange format for the web, supported by all major programming languages, and is both easy for humans to read and write and machines to parse and generate. This specification is designed to be application-independent, and compatible with modern web technologies. For more information on JSON, visit JSON.org.

    A few things to keep in mind:

    • The fields/attributes in an object can be in any order. The id field of the text does not always have to preced the title field, for example.

    Text

    required The text is the fundamental unit of documentation. It represents a linguistic performance of a stretch of discourse in context. This may include a wide variety of linguistic genres: an elicitation session, a conversation, a narrative, a song, etc. 

    
{
  "id" : "346",
  "title" : "Bear and Rabbit",
  "abbreviation" : "A25",
  "participants" : [ { participant 1 }, { participant 2 }, … ],
  "orthographies" : [ { orthography 1 }, { orthography 2 }, … ],
  "phrases" : [ { phrase 1 }, { phrase 2 }, … ]
}

    Text Identifier (ID)

    "id"

    Optional A unique string identifier for the text, (e.g. globally unique identifier, an integer, etc.)

    
"id" : 25

    Title

    "title"

    Required A human-readable prose description of the content of the text.

    
"title" : "Bear and Rabbit"

    Abbreviation

    "abbreviation"

    Optional A human-readable abbreviation for the text, containing only letters a-z, A-Z, and numbers 0-9 (no spaces allowed).

    Participants

    "participants"

    Optional A collection of all the individuals who contributed to this text in some way, for example through recording, digitization, analysis, transcription, or editing.

    
"participants" : [ { participant 1 }, { participant 2 }, ... ]

    Phrases

    "phrases"

    Required A collection of all the Phrases in the Text.

    
  "phrases": [ { phrase 1 }, { phrase 2 }, ... ]

    Phrase

    Required A segmentation of a Text. This standard imposes no requirements regarding how the linguist should implement this segmentation—linguists may choose to use Intonation Units, Turns, Sentences, or any other appropriate subdivision.

    
  {
    "transcriptions": {
      "SwadPhonemic" : "kunugu wey pu·p panš naʔank hi čuyiʔi",
      "ModPhonemic" : "kunugu wey puup panx naqank hi cuyiqi",
      "APAPhonemic" : "kunukʼu wey puːp panš naʔank hi čuyiʔi",
      "ModPractical" : "Kunugu wey puup panx naqank hi cuyiqi.",
      "SwadTranscript" : "kunuguˊ we pu·pˊ panš naʔankˊ hi ču·gšˊ […]"
    },
    
    "translations": {
      "SwadFree" : "The rabbit went to some people […]",
      "Literal" : "It is said the rabbit, going to the peopleʼs place […]"
    }
  }

    Transcriptions

    "transcriptions"

    Required A set of one or more Transcriptions for the Phrase.

    
  "transcriptions" : {
    "Swadesh-phonemic" : "kunugu wey pu·p panš naʔank hi čuyiʔi",
    "Modern-phonemic" : "kunugu wey puup panx naqank hi cuyiqi",
    "APA-phonemic" : "kunukʼu wey puːp panš naʔank hi čuyiʔi"
  }

    Transcription

    [name of orthography]

    Required A phonemic representation of the phrase in an orthography which is either in IPA or can be unambiguously mapped to IPA.

    
              "APA-phonemic" : "kunukʼu wey puːp panš naʔank hi čuːkʼš we kičk heːš ʔučpa ʔučaːšnakš"

    Speaker

    "speaker"

    Optional A unique identifier for the Person who spoke or otherwise created the phrase.

    
  "speaker": "JPH"
